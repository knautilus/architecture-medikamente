## Задание 3. Оценка Data Encryption at Rest and In Transit

### Классификация данных

1. *Персональные*. Персональные данные клиентов: ФИО, паспортные данные, контактные телефоны, адрес проживания, дата рождения, Email.
2. *Медицинские*. Истории болезней, результаты анализов, назначения лечения.
3. *Финансовые*. Договоры с клиентами, данные оплаты услуг, чеки.
4. *Внутренние*. Корпоративная почта, журналы записей к врачам, внутренние акты и распоряжения.
5. *Публичные*. Графики работы специалистов, прайс-листы.

### Инструменты защиты

*Защита данных в состоянии покоя*
- Шифрование данных в БД - pgcrypto для PostgreSQL
- Шифрование данных на устройствах - полное шифрование диска (FDE)
- KMS для управления ключами шифрования - Yandex Key Management Service
- Доступ к данным на основе RBAC/ABAC - Keycloak

*Защита данных в процессе передачи*
- Использование TLS версии 1.2 или выше
- SMTP TLS для корпоративной почты
- Использование хэшей для проверки целостности при передаче файлов

### Реестр средств для защиты данных

| Данные              | В чём необходимость защиты                                             | Инструменты защиты |
| ------------------- | ---------------------------------------------------------------------- | ------------------ |
| Персональные данные | Соответствие требованиям ФЗ-152                                        | Шифрование данных в БД - pgcrypto для PostgreSQL; RBAC/ABAC - Keycloak; аудит доступа; использование TLS при передаче |
| Медицинские данные  | Высокие репутационные потери, необходимость сохранения врачебной тайны | Шифрование данных в БД и на дисках; RBAC/ABAC - Keycloak; аудит доступа; использование TLS при передаче; обезличивание данных |
| Финансовые данные   | Коммерческая тайна                                                     | Шифрование данных в БД; использование TLS при передаче |
| Внутренние          | Коммерческая тайна, репутационные потери                               | Использование TLS при передаче; использование SMTP TLS для корпоративной почты |
